Описание окружения:
- Текущий проект является микрофронтом в spa-приложении.

Подготовка:
1. Если в проекте не установлен `@cloud-ru/ft-config-vitest`, то установи его (используй последнюю доступную версию) в "dependencies".
2. Если в корне проекта отсутствует файл `./vitest.config.ts`, то создай его со следующим содержимым:
```ts
import { default as Config } from '@cloud-ru/ft-config-vitest';

export default Config();
```
3. Если в `package.json` проекта в `scripts` отсутствует команда `test:unit`, то добавь её: `"test:unit": "vitest run"`
4. Если ты выполнил хоть одно из действий выше, напиши "ПОДГОТОВКА VITEST ЗАВЕРШЕНА". Если не пришлось создавать файлы, устанавливать пакет и/или добавлять скрипт, то напиши "ПОДГОТОВКА VITEST НЕ ПОТРЕБОВАЛАСЬ".

Шаги
1. Посмотри на правила, описанные в @tests/unit.mdc и следуй им при выполнении данной команды.
2. По итогу анализа проекта напиши unit тесты на vitest. Запрещено модифицировать существующие файлы!
3. По итогу написания тестов запусти их проверку через npm команду `test:unit`.
4. Считай количество попыток запуска тестов. Если попыток стало больше трёх, то остановись и обратись ко мне за помощью.
5. По итогу прочтения задачи опубликуй сообщение "Я ОЗНАКОМИЛСЯ С УСЛОВИЯМИ ЗАДАЧИ".
6. Проверь, использовал ли ты `any` в качестве типов. Если использовал, то удаляй. Если нечем заменить, то удаляй такой тест.
7. После выполнения всех шагов задай сам себе вопрос: "Я ТОЧНО НЕ НАРУШИЛ НИ ОДНО ИЗ ПРАВИЛ?" и ответь на него.
